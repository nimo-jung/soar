TMS와 SOAR가 융합된 차세대 AI-TMS는 고성능 데이터 파이프라인(대용량 I/O), 웹 플랫폼(멀티테넌시 관리), 그리고 AI 엔진(Agentic AI)이라는 전혀 다른 결의 기술 3가지가 유기적으로 맞물려야 하는 고난도 프로젝트입니다.

따라서 단순히 머릿수만 채우는 것이 아니라, **핵심 설계를 책임질 고급 개발자를 중심으로 초·중급 개발자가 파트를 나누어 병렬로 개발하는 구조**가 필수적입니다.

제품의 프로토타입(MVP) 출시 및 안정적인 초기 상용화 버전 빌드를 기준으로, 각 파트별 중첩 업무를 고려한 **최적의 개발 인원 산정 안**을 제시해 드립니다.

## 🔄 단계별 인력 중첩 및 협업 구조 (WBS 관점)

초기 대수 산정 시 인력이 낭비되지 않도록, 개발 사이클 전반에서 일감이 중첩되는 구역을 다음과 같이 조율해야 합니다.

```
[ 설계 단계 ] -> 고급 2명 중심 (인프라 구조 및 AI 연동 규격 확정)
                ↓
[ 구축 단계 ] -> 중급 3명 + 초급 2명 투입 (NestJS 코어 빌드 및 GoLang 파이프라인 양산)
                ↓
[ AI 융합 ]  -> 백엔드 중급 + AI 중급 (FastAPI - NestJS - RedPanda 정책 배포 연동)
                ↓
[ 화면 연동 ] -> 프론트 초급 + 백엔드 중급 / AI 중급 (대시보드 실시간 탐지 UI 연동)

```

1. **프로젝트 초기 (1~2달차):** 고급 개발자 2명이 ClickHouse, RedPanda의 인프라 규격과 Python AI 엔진 간의 통신 프로토콜을 완벽하게 정의해야 합니다. 이 시기에 중급/초급 개발자들은 NestJS와 React의 보일러플레이트(기본 뼈대 코드)를 구성하고 자체 프로토콜 수집 명세를 선행 분석합니다.
2. **프로젝트 중기 (3~5달차):** 백엔드 중급 개발자는 인프라 고급 개발자와 협업하여 Vector 설정 자동화(Hot Reload)를 뚫어야 하고, 프론트 초급 개발자는 백엔드 중급 개발자가 만들어주는 API를 받아 화면을 붙입니다.
3. **프로젝트 후기 (6달차~):** AI 파트의 중급 개발자가 만든 FastAPI 결과물을 백엔드(NestJS)의 플레이북 엔진과 결합하고, RedPanda 정책 토픽으로 쏘는 **최종 통합 테스트**를 전 직원이 스크럼 형태로 진행합니다.
