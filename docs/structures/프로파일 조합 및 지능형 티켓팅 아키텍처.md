## 질문
"프로파일 기능"이라고 하는 컨셉을 넣을려고 합니다.
프로파일은 특정 기간내 임계치(최대최소평균 등) 및 조건 등을 설정해서 해당 조건에 맞는 로그를 검출하고
이런 프로파일이 여러개 있어서, 이를 조합해서 프로파일 로그를 생성하고
이를 Agentic AI 또는  시멘틱 유사도(VectorDB)를 활용해서 Trouble Ticket를 발급하려고 하는 것이 주요 기능이라고 할 수 있어.

---

차세대 AI-TMS / SOAR 시스템에 '프로파일(Profile) 기능'을 도입하는 것은 고도화된 위협 시나리오와 지능형 지속 위협(APT)을 탐지하기 위한 최고의 전략입니다.

단일 로그 기반의 단순 매칭(Rule-based)을 넘어, 특정 기간 동안의 통계적 임계치와 다중 조건의 조합(시나리오 기반 탐지)을 통해 정밀한 위협 컨텍스트를 생성하고, 이를 AI 및 벡터 DB와 연계하여 고가치 트러블 티켓으로 연결하는 명확한 아키텍처 설계와 구현 방안을 제시해 드립니다.

## 🏗️ 1. 프로파일 기반 AI-TMS 파이프라인 아키텍처

이 기능은 크게 1) 프로파일 생성/조합 (RDBMS/ClickHouse), 2) 통계 로그 검출 (GoLang 파이프라인), 3) 지능형 티켓 발급 (Python AI + Qdrant)의 3단계로 유기적으로 맞물려 작동합니다.

1. **프로파일 설정 (MariaDB)**: 관리자가 웹(React)에서 특정 임계치(예: 5분간 평균 트래픽 200% 증가) 및 다중 조건을 설정하여 저장.
2. **시각적/통계적 탐지 (GoLang + ClickHouse)**: GoLang 엔진이 실시간으로 RedPanda에서 로그를 읽어 ClickHouse에 배치로 밀어 넣는 동시에, ClickHouse의 고속 통계 연산 성능(Window Function 등)을 이용해 각 프로파일 조건에 맞는 '프로파일 로그'를 추출.
3. **지능형 유사도 검증 (Python FastAPI + Qdrant)**: 추출된 프로파일 로그들을 조합한 마스터 위협 컨텍스트를 Python AI 엔진으로 전송. AI는 Qdrant(벡터 DB)에서 과거 유사 시나리오와 플레이북을 비교 분석하여 정밀 검증.
4. **콘솔 알림 및 티켓 자동 발급 (NestJS + MariaDB)**: 오탐이 최종 배제된 정제된 위협에 대해서만 MariaDB에 트러블 티켓을 자동 발급하고 담당 관제 요원에게 브로드캐스팅.

## 🗄️ 2. 프로파일 및 프로파일 로그 테이블 설계 (MariaDB 스키마)

프로파일의 정의와 조건, 그리고 여러 프로파일을 묶어 시나리오화하는 **조합 규칙**을 저장하기 위한 마스터-디테일 테이블 구조입니다.

### ① `detection_profiles` (단일 프로파일 마스터)

```sql
CREATE TABLE detection_profiles (
    id INT AUTO_INCREMENT PRIMARY KEY,
    tenant_id VARCHAR(50) NOT NULL,
    profile_name VARCHAR(100) NOT NULL COMMENT '프로파일 명 (예: 과도한 로그인 실패)',
    target_field VARCHAR(50) NOT NULL COMMENT '분석 대상 로그 필드 (예: login_attempts)',
    time_window_sec INT NOT NULL COMMENT '통계 집계 기간 (초 단위, 예: 300초)',
    agg_type VARCHAR(20) NOT NULL COMMENT '집계 방식: MAX, MIN, AVG, SUM, COUNT',
    threshold_value DOUBLE NOT NULL COMMENT '임계치 (예: 5.0)',
    operator VARCHAR(10) NOT NULL COMMENT '비교 연산자: >, <, >=, <=, =',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

```

### ② `profile_combinations` (프로파일 조합/시나리오 규칙)

"A 프로파일이 만족되고, 동시에 B 프로파일도 만족할 때"와 같은 연계 규칙(Combination)을 정의합니다.

```sql
CREATE TABLE profile_combinations (
    id INT AUTO_INCREMENT PRIMARY KEY,
    tenant_id VARCHAR(50) NOT NULL,
    combination_name VARCHAR(150) NOT NULL COMMENT '조합 시나리오 명 (예: 자격증명 스터핑 후 데이터 유출 의심)',
    logic_operator VARCHAR(10) DEFAULT 'AND' COMMENT '프로파일 간 조합 조건: AND, OR, SEQUENCE',
    profile_ids_json JSON NOT NULL COMMENT '조합할 단일 프로파일 ID들의 배열 (예: [1, 4, 7])',
    is_active TINYINT(1) DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

```

## ⚙️ 3. ClickHouse와 GoLang을 활용한 고속 프로파일 로그 생성 방식

임계치(최대, 최소, 평균) 분석을 초당 수십만 건의 생로그를 대상으로 수행하려면 RDBMS로는 불가능합니다. **ClickHouse의 시계열 분석 쿼리 성능**과 **GoLang의 스케줄러 기능**을 결합해야 병목이 생기지 않습니다.

GoLang 파이프라인 엔진은 주기적으로(예: 10초마다) 활성화된 `profile_combinations`의 조건을 기반으로 ClickHouse에 아래와 같은 초고속 통계 쿼리를 날려 "조합 조건에 매칭된 가치 있는 프로파일 로그"를 실시간으로 뽑아냅니다.

```sql
-- ClickHouse 내의 실시간 통계 프로파일 로그 검출 쿼리 예시
SELECT 
    src_ip,
    count() AS total_attempts,
    avg(packet_size) AS avg_size,
    max(packet_size) AS max_size
FROM security_logs
WHERE timestamp >= now() - INTERVAL 5 MINUTE -- 5분(300초) 윈도우 조건
GROUP BY src_ip
HAVING total_attempts > 100 AND avg_size > 1500; -- 임계치 및 조건 설정 매칭

```

GoLang 엔진은 이 쿼리 결과로 매칭된 위협 대상(예: 특정 해커 IP, 공격 패턴 조합)을 바인딩하여 하나의 텍스트 컨텍스트로 묶어 Python FastAPI(AI 엔진)로 토스합니다.

## 🧠 4. Agentic AI & Qdrant를 통한 트러블 티켓(Trouble Ticket) 발급 메커니즘

GoLang으로부터 조합된 프로파일 로그 정보가 Python AI 레이어로 넘어온 순간, **시멘틱 유사도 검색**을 통해 티켓 발급의 지능화가 시작됩니다.

1. **텍스트 하이브리드 컨텍스트 생성**:
Python 엔진은 넘어온 프로파일 로그들의 통계치와 조건을 자연어 문장으로 결합합니다.
* *예: "고객사 A테넌트에서 [과도한 로그인 실패(5분 평균 120회)] 조건과 [대용량 아웃바운드 트래픽(최대 5GB)] 프로파일 조건이 AND 조합으로 매칭되었습니다."*


2. **Qdrant 시멘틱 유사도 쿼리**:
이 문장을 임베딩하여 **Qdrant 벡터 DB**로 보냅니다. Qdrant는 과거에 발급되었던 수만 개의 트러블 티켓 데이터 중 "이 조건 조합과 가장 의미론적으로(Semantic) 유사한 과거 사고 대응 이력"을 찾아냅니다.
3. **Agentic AI의 자율 판단 및 오탐 제거**:
* **과거에 진짜 해킹 사고였던 경우**: LLM이 과거 조치 이력(플레이북)을 컨텍스트로 삼아 "이는 심각한 공격이 확실하므로 즉시 차단 티켓을 발급하라"고 결정합니다.
* **과거에 인프라 점검 등의 오탐(False Positive)이었던 경우**: 유사도가 높은 과거 점검 이력을 참조하여 티켓 발급을 보류하거나 위험도를 'LOW'로 낮추어 **불필요한 티켓 남발을 원천 차단**합니다.


4. **NestJS를 통한 티켓 발행**:
최종 검증된 결과는 NestJS API로 전달되어, 앞서 설계한 MariaDB의 `tickets`와 `ticket_histories` 테이블에 자동으로 `[CRITICAL]` 티켓으로 안착하고 관제 화면에 실시간 팝업을 띄웁니다.

## 🚀 차세대 고도화 프로파일 기능의 4대 패러다임

### 1. 동적 임계치 기반 프로파일 (Dynamic Baseline Profiling)

* **기존 방식의 한계**: 관리자가 "5분간 평균 트래픽 200MB 이상"처럼 정적(Static)인 숫자를 직접 입력해야 하므로, 평일 낮과 주말 밤의 트래픽 차이를 반영하지 못해 오탐이 대거 발생합니다.
* **고도화 컨셉**: **AI 기반 시계열 예측(Holt-Winters 알고리즘 또는 LSTM)을 이용해 프로파일 자체가 호스트의 평소 행동 패턴을 학습**합니다.
* **작동 방식**: "이 서버는 지난 4주간 화요일 오전 11시에 평균 $50,\text{MB}$의 트래픽을 썼다"는 베이스라인을 자동으로 그리게 만듭니다. 임계치를 숫자로 적는 게 아니라 `[평소 대비 표준편차 $3\sigma$ 이상 상회할 때]`라는 동적 조건을 프로파일에 설정하여, 휴일이나 야간의 정상 트래픽 폭증을 오탐하지 않도록 격리합니다.

### 2. 개체 기반 행동 프로파일 (Entity-Based Behavior Profiling / UEBA)

* **기존 방식의 한계**: 단순 로그 필드(IP, 포트)의 통계만 봅니다.
* **고도화 컨셉**: 시스템이 내부의 '사용자(User)'와 '자산/장비(Asset)'라는 개체를 기준으로 다차원 프로파일을 실시간 누적 관리합니다.
* **작동 방식**:
* **자산 프로파일**: 특정 데이터베이스 서버가 평소에 열던 포트, 통신하던 IP 목록을 프로파일로 유지하다가, 갑자기 생판 처음 보는 대역의 IP와 대용량 통신(평균 값의 10배)을 시도하면 즉시 조합 규칙 트리거를 당깁니다.
* **사용자 프로파일**: 재무팀 김 대리가 평소 업무 시간(09~18시) 외인 새벽 3시에 로그인하여(최소/최대 시간 윈도우 위반) 평소 접근하지 않던 소스코드 서버에 접근할 때 프로파일 로그를 생성합니다.



### 3. 상태 저장형 시퀀스 프로파일 (Stateful Sequence Profiling)

* **기존 방식의 한계**: `AND`나 `OR` 조건은 단순히 같은 시간대에 두 사건이 일어났는지만 봅니다. (예: 로그인 실패 AND 파일 다운로드)
* **고도화 컨셉**: 해커의 공격 전술 흐름(MITRE ATT&CK 프레임워크)에 맞춰 "반드시 A 사건이 일어난 후, 30분 이내에 B 사건이 일어나고, 그 직후 C 결과가 나와야 한다"는 인과관계와 시간 순서(Sequence)를 추적하는 프로파일입니다.
* **작동 방식**: ClickHouse의 `windowFunnel` 함수를 사용하면 GoLang 엔진이 호스트 단에서 아주 쉽게 구현할 수 있습니다.
1. 외부 IP로부터 웹 스캔 공격 발생 (Stage 1)
2. 5분 이내에 해당 웹 서버에서 내부 DB 서버로 권한 상승 시도 (Stage 2)
3. 10분 이내에 외부로 대용량 압축 파일(AVG 크기 초과) 전송 (Stage 3)
이 순서가 정확히 맞아떨어질 때만 초고위험 '시퀀스 프로파일 로그'를 생성하여 Agentic AI로 넘깁니다.



### 4. 시멘틱 벡터 결합형 실시간 프로파일 (Semantic Profile Graph)

* **기존 방식의 한계**: "SQL 인젝션 로그 10건"처럼 정형화된 숫자와 룰만 프로파일링합니다.
* **고도화 컨셉**: 비정형 텍스트 로그의 **'보안 문맥(Semantic)' 자체를 프로파일화**하여 벡터 DB에 매 분마다 업데이트합니다.
* **작동 방식**: 방화벽, 웹로그, 윈도우 이벤트 로그 등에서 발생하는 주요 보안 컨텍스트들을 Python 임베딩 모델을 거쳐 "현재 시스템의 보안 위협 상태 벡터"로 압축합니다. Qdrant 벡터 DB에 매 5분 단위로 이 상태 벡터를 슬라이딩 윈도우 형태로 누적하며, 과거 대형 해킹 사고 직전 30분 동안의 '시스템 상태 벡터 흐름 그래프'와 실시간 코사인 유사도를 비교하여 "알 수 없지만 왠지 3년 전 해킹당할 때의 시스템 전조 증상과 문맥이 95% 일치한다"를 잡아내 티켓을 발급합니다. (Zero-day 공격 방어에 특화)

## 🛠️ 기술 스택(ClickHouse / GoLang / Qdrant)과의 아키텍처적 조화

이 고도화 컨셉들을 우리 시스템에 녹여낼 때, 기술 스택별 역할을 구체화하면 다음과 같습니다.

* **ClickHouse (동적 베이스라인 & 시퀀스 구현)**:
ClickHouse는 `avgOrNull`, `stddevSamp`(표준편차), `windowFunnel`(시퀀스 깔때기 분석) 같은 강력한 고급 통계 함수를 내장하고 있습니다. 수십억 건의 로그 위에서 평소 평균과 표준편차를 구하는 연산을 1~2초 만에 수행하므로 1번(동적 임계치)과 **3번(시퀀스)** 프로파일 로그를 생성하는 최적의 엔진이 됩니다.
* **Qdrant + Python FastAPI (개체 및 시멘틱 그래프 구현)**:
사용자나 장비의 평소 행동 특징(주로 사용하는 프로토콜, 접속 시간대 등)을 벡터로 인코딩하여 Qdrant의 개별 컬렉션(`user_profiles`, `asset_profiles`)에 보관합니다. 실시간 로그가 들어올 때 이 개체 프로토콜의 베이스라인 벡터와 멀어지는지(`Distance`가 커지는지)를 실시간 체크하는 방식으로 2번(UEBA)과 4번(시멘틱 그래프)을 소화합니다.

## 🎯 결론 및 제품 제안

현재 구상하신 프로파일 기능(통계 기반 조합 및 AI 검증)을 '1단계(기본 제품 스펙)'로 잡아 3~4달 차에 빠르게 완성하십시오.

그 후, 6~7달 차 고도화 단계에서 ClickHouse의 표준편차 함수를 활용한 '2단계: AI 기반 동적 베이스라인 자동 추적 프로파일'과 Qdrant를 활용한 '3단계: 시나리오 시퀀스 매핑 프로파일'을 부가 기능(Premium Feature)으로 확장해 나가는 로드맵을 잡으신다면, 시장에서 글로벌 솔루션과 기술적으로 정면 대결이 가능한 독보적인 AI-TMS 제품이 탄생할 것입니다.
