앞서 분석한 "심층 검토 및 미비점 보완 가이드"를 바탕으로, 병목 구간(RestAPI 배포 방식)을 개선하고 AI 레이어(FastAPI, 벡터 DB)를 보완하여 완성한 **차세대 AI-TMS / SOAR 융합 시스템의 최종 확정 기술 스택**입니다.

---

## 🛠️ 차세대 AI-TMS 최종 확정 기술 스택

### 1. 데이터 수집 레이어 (Data Ingestion)

이 기종 및 멀티 테넌트 환경의 대량 로그를 유실 없이 수집하고 표준 포맷으로 전처리하는 레이어입니다.

* **자사 관리 대상 장비**: **자체 수집 프로토콜 엔진**을 활용하여 전용 에이전트 인터페이스 제공.
* **일반 관제 대상 장비**: Vector (by Datadog)를 표준 수집기로 채택. Rust 기반의 초고성능 상태 머신 파서(VRL)를 활용하여 정규표현식 없이 초당 수십만 건의 Syslog를 JSON 구조체로 전처리.

### 2. 메시지 브로커 및 완충 레이어 (Message Broker & Buffer)

트래픽 폭증 시 ClickHouse 적재 지연으로 인한 데이터 유실을 완벽히 차단하는 안전장치입니다.

* **기술 스택**: **RedPanda**
* **채택 이유**: C++ 기반의 단일 바이너리 아키텍처로 ZooKeeper나 JVM 오버헤드가 없어 구조가 가볍고 극도로 빠름. Kafka API와 100% 호환되므로 Vector 및 GoLang과 네이티브 연동 가능. 디스크 지속성을 보장하여 로그 대기열을 안전하게 보호.

### 3. 데이터 파이프라인 레이어 (Data Pipeline)

큐에 쌓인 대량의 전처리 데이터를 ClickHouse로 안전하게 대량 적재(Batch Insert)하는 핵심 엔진입니다.

* **기술 스택**: **GoLang (Goroutine & Channel)**
* **채택 이유**: C++ 대비 압도적인 개발 속도 및 유지보수성 확보. 고루틴을 통해 RedPanda의 토픽 데이터를 수천 건 단위의 청크(Chunk)로 묶어 ClickHouse가 가장 선호하는 배치 형태로 인서트 성능 극대화.

### 4. 데이터 스토리지 레이어 (Database / Polyglot Storage)

데이터의 특성(시계열 로그 vs 관계형 관리 데이터)에 따라 스토리지를 분리하여 효율을 극대화한 구조입니다.

* **로그 및 분석 DB**: **ClickHouse (ReplicatedMergeTree)**
* 초당 수십만 건의 대량 로그 배치 적재와 대규모 다차원 분석(Select) 연산을 SIMD 하드웨어 병렬 가속으로 초고속 처리.


* **관리 및 영속성 DB**: **MariaDB**
* 테넌트 정보, 웹 사용자 계정, 시스템 설정, 장비 토폴로지 등 전통적인 RDBMS의 트랜잭션과 무결성이 필요한 핵심 관리 데이터 저장.



### 5. 백엔드 및 오케스트레이션 레이어 (Control & Orchestration)

SOAR 시스템의 비즈니스 로직, 멀티테넌트 격리 관리, 정책 제어를 담당하는 마스터 레이어입니다.

* **기술 스택**: **NestJS (TypeScript)**
* **채택 이유**: 모듈화가 뛰어난 엔터프라이즈 아키텍처. 런타임에 테넌트 식별자(`tenant_id`)를 확인하여 ClickHouse의 물리적 데이터베이스를 동적으로 격리/라우팅하는 멀티테넌시 구조를 유연하게 제어.

### 6. 프론트엔드 레이어 (User Interface)

보안 관제 요원 및 관리자에게 실시간 위협 상황과 대시보드를 제공하는 대화형 UI입니다.

* **기술 스택**: **React + PrimeReact**
* **채택 이유**: 대규모 컴포넌트 상태 관리 유연성. 실시간 보안 관제 화면(SIEM/TMS) 및 플레이북 흐름도 시각화를 위한 컴포넌트 에코시스템 풍부.

### 7. AI 분석 및 Agentic AI 레이어 (Intelligence Engine)

SOAR의 판단 영역을 자율형 AI 요원(Agentic AI) 형태로 고도화하는 지능형 서브시스템입니다.

* **AI 런타임**: **Python (LangChain / CrewAI / PyTorch)**
* 복잡한 보안 컨텍스트 해석, 플레이북 자동 생성, 이상 징후 탐지를 위한 표준 AI 생태계 활용.


* **인터페이스 API**: **FastAPI**
* GoLang 파이프라인이나 NestJS가 AI 엔진을 비동기로 빠르게 호출하여 탐지/인증 결과를 주고받을 수 있는 초경량 고성능 비동기 API 서버 구현.


* **컨텍스트 레퍼런스 DB**: **초경량 Vector DB (예: Qdrant 또는 pgvector)**
* 과거 위협 침해 지표(IoC), 유사 대응 이력, 위협 인텔리전스(TI) 임베딩 데이터를 보관하여 AI 에이전트의 실시간 RAG(검색 증강 생성) 컨텍스트 검색 성능 보장.



### 8. 위협 대응 및 정책 배포 레이어 (Response & Policy Distribution) ✨ *핵심 변경 구간*

AI가 내린 판단을 기반으로 네트워크 차단 정책을 유관 장비에 실시간 전파하는 자동화 메커니즘입니다.

* **자사 관리 대상 장비**: **자체 정책 배포 프로토콜**을 사용하여 실시간 동기화.
* **일반 관제 대상 장비**: **RedPanda 이벤트 기반 Push 구조 활용 + Vector API**
* **[수정 사항]** 기존의 수동적인 RestAPI(Pull) 방식에서 발생하는 트래픽 병목을 완벽히 차단. NestJS가 차단 IP 목록(IoC)을 RedPanda의 특정 정책 토픽에 발행(Publish)하면, 수집단에 위치한 Vector(또는 타사 장비 에이전트)가 이를 실시간 구독(Subscribe)하여 방화벽/네트워크 장비의 차단 룰셋에 즉시 반영.
