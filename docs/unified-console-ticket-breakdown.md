# Unified Console 티켓 분해본

작성일: 2026-06-05  
기준 문서: docs/unified-console-adr.md

## 1. 운영 원칙

1. Admin/Tenant API 경계는 유지한다.
2. 메뉴 표시와 API 권한은 분리한다.
3. 모든 CUD 및 act-as 이벤트는 감사로그를 기록한다.
4. 멀티테넌트 ON/OFF 회귀 테스트를 동일 비중으로 운영한다.

## 2. Epic 구성

1. EPIC-U1: 통합 Shell 기반 구축
2. EPIC-U2: 인증/권한/토큰 교환 흐름 구축
3. EPIC-U3: 메뉴 정책 및 UI 동작 완성
4. EPIC-U4: 보안/품질 검증 및 릴리즈

## 3. 티켓 백로그

## EPIC-U1: 통합 Shell 기반 구축

### U1-01. Unified Console 진입 라우트 설계
- 목표: 단일 진입점에서 권한 기반 메뉴 렌더링 준비
- 작업 내용:
  1. /console 진입 라우트 정의
  2. 상단 프레임/좌측 메뉴/콘텐츠 슬롯 레이아웃 구성
  3. ON/OFF 모드별 기본 랜딩 정책 정의
- 수용 기준:
  1. 로그인 사용자에 따라 /console 접속 가능
  2. 비로그인 사용자는 로그인 화면으로 리다이렉트
- 예상 공수: 1d

### U1-02. 공통 네비게이션 모델 정의
- 목표: 메뉴 권한 매핑을 코드/문서로 표준화
- 작업 내용:
  1. MenuItem 스키마(키, 라벨, 경로, 필요 권한, 모드 조건) 정의
  2. 시스템 설정 메뉴군/서비스 메뉴군 분류
  3. i18n 키 매핑 반영
- 수용 기준:
  1. 메뉴 데이터만으로 렌더링 가능
  2. 권한/모드 변경 시 코드 수정 최소화
- 예상 공수: 0.5d

## EPIC-U2: 인증/권한/토큰 교환 흐름 구축

### U2-01. 공개 모드 조회 API 추가
- 목표: 프론트가 ON/OFF 상태를 즉시 반영
- 작업 내용:
  1. GET /auth/tenant/mode (isMultiTenantEnabled) 추가
  2. 캐시 정책 및 에러 응답 규격 정의
  3. API 문서 반영
- 수용 기준:
  1. 모드 토글 후 프론트 새로고침 시 상태 즉시 일치
- 예상 공수: 0.5d

### U2-02. Master -> system tenant act-as 토큰 교환 API
- 목표: OFF 모드에서 Master가 서비스 메뉴 접근 가능하게 함
- 작업 내용:
  1. POST /auth/tenant/act-as/system 엔드포인트 추가
  2. Master 권한 검증 후 system tenant 전용 단기 토큰 발급
  3. jti, TTL, 세션 폐기 정책 적용
  4. 발급/실패/종료 감사로그 기록
- 수용 기준:
  1. OFF 모드 Master 계정만 발급 가능
  2. ON 모드에서는 기본 차단(정책 예외 시 별도 플래그)
  3. 감사로그 누락 0건
- 예상 공수: 2d

### U2-03. 백엔드 권한 매트릭스 점검
- 목표: 메뉴 노출 변화와 무관하게 API 보안 유지
- 작업 내용:
  1. 주요 Tenant API에 RolesGuard 정책 재검토
  2. Master 토큰 직접 접근 차단 재확인
  3. 테스트 케이스 보강
- 수용 기준:
  1. 허용되지 않은 role 접근 403 유지
  2. 토큰 타입 혼용 접근 차단
- 예상 공수: 1d

## EPIC-U3: 메뉴 정책 및 UI 동작 완성

### U3-01. 멀티테넌트 ON 메뉴 정책 구현
- 목표: 기존 분리 UX를 통합 Shell에서도 동일 유지
- 작업 내용:
  1. Master: 시스템 설정 메뉴만 노출
  2. Tenant: 서비스 메뉴만 노출
  3. 잘못된 직접 URL 접근 시 권한 오류 처리
- 수용 기준:
  1. ON 모드에서 역할별 메뉴 분리 100%
- 예상 공수: 1d

### U3-02. 멀티테넌트 OFF 단일 뷰 메뉴 구현
- 목표: 최고관리자 단일 화면에서 양쪽 메뉴 동시 사용
- 작업 내용:
  1. OFF 모드 + Master 로그인 시 시스템/서비스 메뉴 동시 노출
  2. 서비스 메뉴 진입 시 act-as 토큰 자동 교환
  3. 토큰 만료/교환 실패 UX 처리
- 수용 기준:
  1. OFF 모드에서 별도 재로그인 없이 양쪽 메뉴 전환 가능
  2. 오류 시 복구 동선 제공
- 예상 공수: 1.5d

### U3-03. 로그인/부트스트랩/비밀번호 재설정 단일테넌트 UX 개선
- 목표: OFF 모드에서 tenantSlug 입력 부담 제거
- 작업 내용:
  1. OFF 모드에서 tenantSlug 입력 숨김
  2. 내부 전송값 system 고정
  3. 부트스트랩/리셋 링크 파라미터 자동화
- 수용 기준:
  1. OFF 모드에서 tenantSlug 수기 입력 없이 인증 플로우 완료
- 예상 공수: 1d

## EPIC-U4: 보안/품질 검증 및 릴리즈

### U4-01. 감사로그 검증
- 목표: 신규 인증/권한 변경의 감사 추적성 보장
- 작업 내용:
  1. act-as 발급/사용/종료 action 코드 정의
  2. 로그 필드(actor/action/resource/tenant/time) 검증
  3. 누락 케이스 보완
- 수용 기준:
  1. 신규 액션 감사로그 누락 0건
- 예상 공수: 0.5d

### U4-02. ON/OFF 통합 스모크 테스트 추가
- 목표: 모드 전환 회귀를 CI에서 조기 탐지
- 작업 내용:
  1. ON 시나리오: 메뉴 분리, 권한 차단, 정상 로그인
  2. OFF 시나리오: 통합 메뉴 노출, 서비스 메뉴 접근, act-as 성공
  3. 부트스트랩/비밀번호 재설정 시나리오 포함
- 수용 기준:
  1. CI에서 ON/OFF 핵심 플로우 자동 검증
- 예상 공수: 1d

### U4-03. 운영/릴리즈 가이드 업데이트
- 목표: 운영자가 모드별 동작을 명확히 이해
- 작업 내용:
  1. ON/OFF 모드별 UX 차이 문서화
  2. 장애 대응(runbook) 업데이트
  3. 롤백 절차 문서화
- 수용 기준:
  1. 운영 문서 리뷰 완료
- 예상 공수: 0.5d
